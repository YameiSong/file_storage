Data Analysis for Congressional Tweets¶

Author: Yamei Song

I get the congressional tweets dataset from Lobbyists4America. I take Lobbyists4America as my hypothetical client. In order to help my client focus and strengthen his/her lobbying efforts, I will find the key topics, members, and relationships within Congress from this dataset.

ERD¶

Hypothesis¶

  1. What are the key topics of those congressional tweets and how often?

    Can be inferred from tweet texts.

    Key topics maybe health, education, technology, and alike.

  1. Who publishes congressional tweets and how often?

    Will count a user’s tweets in each year.

    Maybe the time to publish a tweet affects its favourites.

    Maybe the more frequent a user publishes tweets, the more average favourites he/she gets.

  1. What are the relationships between people who publish congressional tweets?

    Users will be divided into clusters according to their political views.

Data Analysis Approach¶

  • Will extract entities from tweet texts and look at entity count/frequency

    For topic analysis.

  • Will look at frequency metrics

    For frequency of posting tweets during a time period.

  • Will use machine learning models

    For sentiment analysis of tweet texts.

    For clustering like-minded tweeters.

def sentiment_analysis(text):
    
    # return a tuple (polarity, subjectivity)
    
    blob = TextBlob(text)
    
    return (blob.sentiment.polarity, blob.sentiment.subjectivity)

Theme extraction¶

Since each tweet is pretty short and lacks information, themes are extracted from all tweets of each user.

Steps:

  1. Group the tweets by user_id.
  2. Combine all tweet texts of one user together.
  3. Clean the text by removing some signs (#'"/).
  4. Use TextBlob to extract entities from texts.

In this way, I get the themes that a user is mostly interested in.

In [21]:
from textblob.np_extractors import ConllExtractor
import collections

def extract_themes(text):
    
    # extract entities
    extractor = ConllExtractor()
    blob = TextBlob(text, np_extractor=extractor)
    entities = list(blob.noun_phrases)
    
    # take top 10 entities as themes
    counter = collections.Counter(entities)
    themes = list(list(zip(*counter.most_common(10)))[0])
    return themes

import re

def clean_text(text):
    text = re.sub(r'[^,.@!a-zA-Z\s]', '', text)
    return text

Extract text features¶

Extract these features from users' tweets:

  • sentiment of theme: polarity, subjectivity
  • frequency of theme: count
In [53]:
themes = [
    'obamacare',
    'trump',
    'health',
    'medi',
    'russia',
    'israel',
    'iran',
    'tax',
    'climate',
]

df_features = pd.DataFrame()

for t in themes:
    # filter rows that contains 't' in the text
    df = df_tweets[
        df_tweets['text'].str.contains(t, case=False, regex=True)
    ].copy(deep=True)
    
    df.drop(
        ['created_at', 'favorite_count', 'id', 'retweet_count', 'text'], 
        axis=1,
        inplace=True
    )
    
    # calculate mean polarity, mean subjectivity, and count
    df_agg = df.groupby(['user_id']).mean()
    df_agg['count'] = df.groupby(['user_id'])['polarity'].count()
    
    df_agg.rename(
        columns={
            'polarity': 'polarity_' + t,
            'subjectivity': 'subjectivity_' + t,
            'count': 'count_' + t
        },
        inplace=True
    )
    
    df_agg.reset_index(inplace=True)
    
    if df_features.empty:
        df_features = df_agg
    else:
        df_features = df_features.merge(df_agg, how='outer', on='user_id')

# add '(user) name' column
df_user_features = df_features.merge(df_users[['id', 'name']], how='left', left_on='user_id', right_on='id')
df_user_features.drop('id', axis=1, inplace=True)

K-Means Clustering¶

Determine the optimal number of clusters¶

Use the elbow method:

  1. Compute clustering algorithm (e.g., k-means clustering) for different values of k. For instance, by varying k from 1 to 10 clusters.
  2. For each k, calculate the error sum of squares (SSE).
  3. Plot the curve of SSE according to the number of clusters k.
  4. The location of a bend (knee) in the plot is generally considered as an indicator of the appropriate number of clusters.
In [56]:
from sklearn.cluster import KMeans

sse = dict()

data = df_user_features.drop(['user_id', 'name'], axis=1).fillna(0)

for k in range(2, 10):
    kmeans = KMeans(n_clusters=k, max_iter=1000).fit(data)
    sse[k] = kmeans.inertia_

plt.figure()
plt.plot(list(sse.keys()), list(sse.values()))
plt.xlabel("Number of cluster")
plt.ylabel("SSE")
plt.show()

The picture above indicates that the optimal number of clusters is 4.
